Self-Service Infrastructure Platform

Lets developers request their own environments using Jenkins (pipeline), Terraform (provisioning), and Ansible (configuration).

Jenkins Terraform Ansible Python (Flask) JCasC

GitHub

Repository Layout & Components

  • terraform/ — infrastructure modules and definitions
  • ansible/ — post-provision configuration playbooks
  • jenkins/ — pipeline definitions (Jenkinsfile, JCasC)
  • frontend/ — simple self-service UI
  • status/ — operation/status tracking
  • General overview and license are in the README.

Key Features

  • Parameterized pipeline for create / destroy flows
  • Dynamic Ansible inventory generated from Terraform outputs
  • Jenkins Configuration as Code (JCasC) for automated setup
  • Request environments via a simple web interface

Example Flow

# Prepare environment variables
./generate_env.sh

# Provision resources with Terraform
cd terraform
terraform init && terraform apply -auto-approve

# Generate dynamic inventory and run Ansible
../generate_inventory.sh
cd ../ansible
ansible-playbook site.yml

# Tear down when finished
cd ../terraform
terraform destroy -auto-approve

Exact commands may vary by environment. The same flow is also embedded in the Jenkins pipeline.
