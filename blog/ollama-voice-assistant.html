Ollama Voice Assistant

Runs on local models (Ollama), supports persona selection, and is automated end-to-end with a Jenkins pipeline (Web UI + API + optional TTS/STT).

Ollama FastAPI Docker Compose Jenkins Web UI

GitHub

Highlights

  • Local LLM (Ollama) + persona selection (Customer Support, DevOps Advisor, Technical Coach).
  • Text & voice interaction via a simple web UI; optional TTS (e.g., pyttsx3) and STT.
  • Parameterized Jenkins pipeline: pulls model, boots services, health-checks, smoke tests, and logs.
  • Single-command setup with Docker Compose.

Project Structure

agents/             # FastAPI backend

See the repository README for details.

Setup & Run

Option A — With Jenkins (recommended)

docker compose -f docker/docker-compose.jenkins.yml up -d --build
# Jenkins: http://localhost:8081  (admin via JCasC)
# "Build with Parameters" → choose model, persona, directive

Option B — Local

docker compose -f docker/docker-compose.yml up -d --build
# Ollama API:  http://localhost:11434
# Agent API:   http://localhost:8080
# Web UI:      http://localhost:8080/ui/

Health Checks

curl -s http://localhost:8080/healthz
curl -s http://localhost:8080/readyz

Environment variables and additional instructions are documented in the README.

API Example

curl -s http://localhost:8080/chat -H 'Content-Type: application/json' -d '{
  "persona":"DevOps Advisor",
  "directive":"Be concise and accurate.",
  "message":"How do I roll back a bad deploy?",
  "model":"phi3:mini",
  "speak": false
}'

Example responses and pipeline steps are provided in the repository.
